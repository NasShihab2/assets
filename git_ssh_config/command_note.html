Technical Guide

Multi-Account SSH

A step-by-step guide to managing multiple GitHub accounts on a single machine using SSH keys.

1
Generate Key for First Account

Run this command to generate your primary SSH key. Replace the email with your first GitHub account email.

ssh-keygen -t ed25519 -C "your_email@example.com" -f C:\Users\Admin\.ssh\id_ed25519
Press Enter for defaults. Adding a passphrase is optional but recommended.

2
Add First Key to GitHub

Open the public key file and copy the content to your GitHub settings.

notepad C:\Users\Admin\.ssh\id_ed25519.pub
Go to Settings → SSH and GPG keys → New SSH key and paste the content.

3
Generate Key for Second Account

Generate a separate key with a different filename for your secondary account.

ssh-keygen -t ed25519 -C "second_email@example.com" -f C:\Users\Admin\.ssh\id_ed25519_second

4
Add Second Key to GitHub

Repeat the process for your second GitHub account using the second public key.

notepad C:\Users\Admin\.ssh\id_ed25519_second.pub

5
Configure SSH Config File

Now, create or edit the SSH config file to tell SSH which key to use for which host.

notepad C:\Users\Admin\.ssh\config

Paste the following configuration:

# First GitHub account
Host github.com-first
    HostName github.com
    User git
    IdentityFile C:\Users\Admin\.ssh\id_ed25519

# Second GitHub account
Host github.com-second
    HostName github.com
    User git
    IdentityFile C:\Users\Admin\.ssh\id_ed25519_second

6
Test Connections

Verify that both accounts are successfully authenticated.

ssh -T git@github.com-first ssh -T git@github.com-second
Successful auth returns: "Hi username! You've successfully authenticated..."

7
Clone or Update Repos

When cloning or setting remotes, use the custom hostnames defined in your config.

# Clone with first account git clone git@github.com-first:username/repo.git # Clone with second account git clone git@github.com-second:username/repo.git

8
Force SSH Globally (Optional)

Automatically redirect all HTTPS clones to SSH to avoid credential manager issues.

git config --global url."git@github.com:".insteadOf "https://github.com/"
